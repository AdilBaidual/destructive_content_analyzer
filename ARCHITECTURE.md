## Модели машинного обучения

### 1. Модель деструктива (SGDClassifier)

**Назначение:** Классификация текста на деструктивный/нейтральный.

**Архитектура:**
- **Алгоритм:** SGDClassifier с функцией потерь `log_loss` (эквивалент логистической регрессии)
- **Векторизация:** TF-IDF (Term Frequency - Inverse Document Frequency)
- **Размер словаря:** 5000 признаков

**Формула TF-IDF:**
```
TF-IDF(t,d) = TF(t,d) × IDF(t)

где:
- TF(t,d) = частота термина t в документе d
- IDF(t) = log(N / df(t))
- N = общее количество документов
- df(t) = количество документов, содержащих термин t
```

**Почему SGDClassifier:**
- Поддерживает инкрементальное обучение (`partial_fit`)
- Эффективен на больших датасетах
- Хорошо работает с разреженными данными (TF-IDF)

**Гиперпараметры:**
```python
SGDClassifier(
    loss='log_loss',        # Логистическая регрессия
    max_iter=1000,          # Максимум итераций
    class_weight='balanced' # Балансировка классов
)
```

**Датасет:** `toxic_comments.csv` (~28 000 примеров)

### 2. Модель экстремизма (Keras Sequential)

**Назначение:** Классификация текста на экстремистский/нейтральный.

**Архитектура нейронной сети:**

```
Input (последовательность индексов слов)
        │
        ▼
┌───────────────────────────────┐
│  Embedding Layer              │
│  input_dim=1000, output_dim=16│
│  Преобразует слова в векторы  │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│  GlobalAveragePooling1D       │
│  Усреднение векторов слов     │
│  (seq_len, 16) → (16,)        │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│  Dense(16, activation='relu') │
│  Скрытый слой                 │
└───────────────┬───────────────┘
                │
                ▼
┌───────────────────────────────┐
│  Dense(1, activation='sigmoid')│
│  Выход: вероятность [0, 1]    │
└───────────────────────────────┘
```

**Принцип работы слоёв:**

1. **Embedding Layer:** Каждое слово (представленное индексом) преобразуется в плотный вектор размерности 16. Это позволяет модели изучать семантические отношения между словами.

2. **GlobalAveragePooling1D:** Усредняет все векторы слов в предложении, создавая единый вектор фиксированной размерности. Это делает модель независимой от длины входного текста.

3. **Dense(16, relu):** Полносвязный слой с функцией активации ReLU для нелинейного преобразования признаков.

4. **Dense(1, sigmoid):** Выходной слой с сигмоидой для получения вероятности принадлежности к классу экстремизма.

**Функция потерь:** Binary Cross-Entropy
```
L = -[y·log(p) + (1-y)·log(1-p)]
```

**Оптимизатор:** Adam (lr=0.01)

**Датасет:** `extremism_data.csv` (~5 700 примеров)

## Процесс обработки данных

### Этап 1: Парсинг

Модуль `tg_parser.py` использует библиотеку Telethon для получения данных из Telegram API.

**Собираемые данные:**
- `text` — текст поста
- `created_at` — дата публикации
- `post_id` — идентификатор поста

### Этап 2: Предобработка

Модуль `preprocessor.py` выполняет очистку текста:

```python
def preprocess_text(text):
    text = text.lower()                    # Нижний регистр
    text = remove_emoji(text)              # Удаление эмодзи
    text = remove_urls(text)               # Удаление ссылок
    text = remove_mentions(text)           # Удаление @mentions
    text = remove_hashtags(text)           # Удаление #hashtags
    text = remove_special_chars(text)      # Удаление спецсимволов
    text = remove_stopwords(text)          # Удаление стоп-слов
    return text
```

**Пример:**
```
До:  "Привет @user! Смотри https://link.com 🔥 #новости"
После: "привет смотри новости"
```

### Этап 3: Анализ

1. Загрузка подготовленных данных
2. Применение модели деструктива
3. Вычисление статистики
4. Генерация визуализаций

## Механизм дообучения

### Для модели деструктива (SGDClassifier)

Используется метод `partial_fit()` — инкрементальное обучение без полного переобучения.

```python
def retrain_destructive_model(new_texts, new_labels):
    model, vectorizer = load_model()

    # Векторизация новых данных
    X_new = vectorizer.transform(new_texts)

    # Инкрементальное обучение
    model.partial_fit(X_new, new_labels)

    # Сохранение обновлённой модели
    save_model(model)
```

**Преимущества:**
- Не требует хранения всего датасета в памяти
- Быстрое обновление модели
- Сохранение ранее изученных паттернов

### Для модели экстремизма (Keras)

Используется стандартный `model.fit()` на новых данных:

```python
def retrain_extremism_model(new_texts, new_labels):
    model, tokenizer, maxlen = load_model()

    # Токенизация
    X_new = tokenizer.texts_to_sequences(new_texts)
    X_new = pad_sequences(X_new, maxlen=maxlen)

    # Дообучение (fine-tuning)
    model.fit(X_new, new_labels, epochs=5)

    save_model(model)
```

### Сохранение новых примеров

Все новые размеченные примеры сохраняются в дополнительные датасеты:
- `additional_destructive.csv`
- `additional_extremism.csv`

При следующем полном обучении эти данные объединяются с основным датасетом.

## Веб-интерфейс

### Технологии
- **Backend:** Flask (Python)
- **Frontend:** HTML, CSS (встроенные стили)
- **Шаблонизатор:** Jinja2

### Страницы

1. **Главная (/)** — анализ TG-канала
   - Ввод названия канала
   - Количество постов
   - Отображение результатов

2. **Дообучение (/retrain)** — три шага:
   - Ввод канала и ID постов
   - Предпросмотр и разметка
   - Запуск дообучения

### Визуализации

- **Распределение классов** — столбчатая диаграмма
- **Гистограмма вероятностей** — распределение предсказаний
- **Timeline** — динамика по датам
- **WordCloud** — облако частых слов
- **Топ постов** — посты с максимальной вероятностью
